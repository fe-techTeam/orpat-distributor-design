## Naming Conventions

**Files:**
- Main file: `index.html` (lowercase)
- Images: `kebab-case.ext` (e.g., `fan-bldc-white.jpg`)
- API spec: Title case with spaces (Postman standard)

**Directories:**
- Lowercase: `images/`, `.planning/`, `.claude/`
- Hidden directories: `.git/`, `.claude/`, `.planning/`

**CSS Classes:**
- kebab-case: `.product-card`, `.checkout-item`, `.btn-add`
- Component pattern: `{component}-{element}` (e.g., `.header-back`, `.cart-text`)
- State modifiers: `.active`, `.pending`, `.approved`, `.dispatched`

**Screen Labels:**
- Format: `{Letter} - {Description}`
- Examples: `A - Home`, `E - Checkout`, `K - Orders`
- Range: A through W (21 screens)

## Where to Add New Code

**New Screen:**
- Location: Add new `.phone-frame` block in `index.html`
- Pattern: Follow existing screen structure with `.screen` wrapper
- Label: Add `<span class="screen-label">X - Name</span>`

**New CSS Styles:**
- Location: `index.html` in `<style>` block (lines 10-1269)
- Pattern: Add new class definitions following existing patterns
- Variables: Use `var(--orpat-red)` etc. for brand colors

**New Images:**
- Location: `images/` directory
- Naming: `{product}-{variant}.{ext}`
- Format: JPG for photos, PNG for transparency

**API Endpoints:**
- Location: `Distributor APP.postman_collection.json`
- Pattern: Add to `item` array following existing structure
